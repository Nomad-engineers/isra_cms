# Примеры

Payload предоставляет широкий спектр примеров, которые помогут вам начать работу с вашим проектом, независимо от того, чем вы занимаетесь. Эти примеры разработаны так, чтобы их было легко запустить и понять. Они демонстрируют не более чем конкретные функции, которые демонстрируются, чтобы вы могли легко понять, что именно происходит.

## Доступные примеры

### [Auth](https://github.com/payloadcms/payload/tree/main/examples/auth)
Пример аутентификации пользователей, демонстрирующий:
- Настройку аутентификации
- Защищенные маршруты
- Регистрация и вход пользователей
- Управление сессиями

### [Custom Components](https://github.com/payloadcms/payload/tree/main/examples/custom-components)
Пример создания пользовательских компонентов для административной панели:
- Кастомные поля
- Кастомные представления
- Стилизация компонентов
- Интеграция с React

### [Draft Preview](https://github.com/payloadcms/payload/tree/main/examples/draft-preview)
Пример предварительного просмотра черновиков:
- Функциональность черновиков
- Предварительный просмотр перед публикацией
- Интеграция с фронтендом

### [Email](https://github.com/payloadcms/payload/tree/main/examples/email)
Пример отправки email уведомлений:
- Настройка email сервисов
- Триггеры на основе событий
- Шаблоны email

### [Form Builder](https://github.com/payloadcms/payload/tree/main/examples/form-builder)
Пример конструктора форм:
- Динамическое создание форм
- Валидация форм
- Обработка отправленных данных

### [Live Preview](https://github.com/payloadcms/payload/tree/main/examples/live-preview)
Пример живого предпросмотра контента:
- Редактирование в реальном времени
- Синхронизация с фронтендом
- WebSocket соединения

### [Multi-tenant](https://github.com/payloadcms/payload/tree/main/examples/multi-tenant)
Пример мульти-тенантной архитектуры:
- Разделение данных по клиентам
- Контроль доступа на уровне организации
- Изоляция данных

### [Tailwind / Shadcn-ui](https://github.com/payloadcms/payload/tree/main/examples/tailwind-shadcn-ui)
Пример интеграции с Tailwind CSS и Shadcn-ui:
- Стилизация административной панели
- Кастомные UI компоненты
- Responsive дизайн

### [White-label Admin UI](https://github.com/payloadcms/payload/tree/main/examples/whitelabel)
Пример белого брендинга административной панели:
- Кастомизация логотипа и цветов
- Удаление брендинга Payload
- Полная кастомизация интерфейса

## Как запустить примеры

Если вы хотите запустить примеры, вы можете использовать `create-payload-app` для создания проекта из одного из них:

```bash
npx create-payload-app --example example_name
```

Замените `example_name` на название конкретного примера, который вы хотите использовать.
